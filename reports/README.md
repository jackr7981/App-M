# Model Availability Reports

This directory contains automated reports from the Gemini model availability checker.

## Purpose

Track the availability of Google Gemini AI models over time to:
- Ensure the app's AI features remain operational
- Detect when models are deprecated or removed
- Provide historical data for model stability analysis

## Report Files

### Daily Reports
- **Format**: `gemini-model-check-YYYY-MM-DD.json`
- **Retention**: Indefinite (tracked in Git)
- **Content**: Complete snapshot of model availability for that day

### Latest Report
- **File**: `latest-model-check.json`
- **Purpose**: Always contains the most recent check results
- **Usage**: Referenced by dashboards and monitoring tools

## Report Structure

```json
{
  "timestamp": "2026-02-14T00:00:00.123456",
  "primary_model": "models/gemini-2.5-flash",
  "primary_model_available": true,
  "total_models": 15,
  "gemini_models": [
    "models/gemini-1.5-flash",
    "models/gemini-2.5-flash",
    "..."
  ],
  "fallback_models": {
    "configured": ["models/gemini-2.0-flash-exp", "..."],
    "available": ["models/gemini-2.0-flash-exp", "..."]
  },
  "status": "OK"
}
```

## Status Values

- **OK**: Primary model is available
- **WARNING**: Primary model unavailable, fallbacks available
- **CRITICAL**: Primary model and all fallbacks unavailable

## Automated Generation

Reports are automatically generated by:
- **GitHub Actions**: `.github/workflows/gemini-model-check.yml`
- **Script**: `scripts/check_gemini_models.py`
- **Schedule**: Daily at 00:00 UTC

## Viewing Reports

### Via GitHub
1. Navigate to repository
2. Browse to `/reports` folder
3. Click on any JSON file
4. Use GitHub's JSON viewer

### Via GitHub Actions
1. Go to **Actions** tab
2. Select **Gemini Model Availability Check**
3. Click on latest workflow run
4. View **Summary** for formatted report
5. Download **Artifacts** for all reports

### Locally
```bash
# View latest report
cat reports/latest-model-check.json | jq

# View specific date
cat reports/gemini-model-check-2026-02-14.json | jq

# Check primary model status
jq '.primary_model_available' reports/latest-model-check.json
```

## Historical Analysis

To analyze model stability over time:

```bash
# List all available models on a specific date
jq '.gemini_models' reports/gemini-model-check-2026-02-14.json

# Check if model was ever unavailable
grep -r '"primary_model_available": false' reports/

# Count total models over time
for file in reports/gemini-model-check-*.json; do
  echo "$file: $(jq '.total_models' $file)"
done
```

## Cleanup Policy

- **Keep all reports**: For historical tracking
- **Archive older than 1 year**: Consider moving to separate branch (optional)
- **Never delete**: Model availability data is valuable for debugging past incidents

## Alert Thresholds

The monitoring system triggers alerts when:

| Condition | Alert Level | Action |
|-----------|-------------|--------|
| Primary model unavailable | WARNING | Email notification, recommend fallback |
| All fallback models unavailable | CRITICAL | Immediate attention required |
| API connection failure | ERROR | Check API key and service status |

## Integration

These reports can be integrated with:

- **Monitoring dashboards**: Parse JSON for real-time status display
- **Alerting systems**: Send notifications via Slack, Discord, etc.
- **Analytics tools**: Track model adoption and deprecation trends
- **Documentation**: Reference in incident reports and post-mortems
